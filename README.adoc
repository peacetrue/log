== 概述
//日志呢，是用于核对事实真相的。
日志组件用于记录日志信息，日志信息包括操作编码、操作描述、操作人、操作时间、模块编码、记录标识、输入参数、输出结果、异常信息、操作耗时。 其中，操作编码、操作描述、操作人、操作时间是基本信息；模块编码、记录标识用于确定日志所归属的记录，比如说某一个用户下会有注册、登录、注销各种日志记录； 输入参数、输出结果、异常信息、操作耗时属于具体的调试信息，当系统出现异常时，可以方便定位问题。

== 要求

* ``jdk1.8``及以上
* ``spring-boot:1.5.20.RELEASE``及以上，暂不支持``spring-boot:2.0.0.RELEASE``及以上
* 持久层使用mybatis实现，暂未提供Hibernate等其他ORM框架的实现，mybatis版本为``3.4.6``

=== 日志服务

然后引用最基本的类库：

.日志服务的mybatis实现
[source%nowrap,maven]
----
<dependency>
    <groupId>com.github.peacetrue</groupId>
    <artifactId>peacetrue-log-service-mybatis</artifactId>
    <version>2.0-SNAPSHOT</version>
</dependency>
----

'peacetrue-log-service-mybatis/schema'目录下提供了mybatis的建表脚本， 包括**schema-mysql-lsbbbs.sql**、**schema-mysql-lsssss.sql** 。 **lsbbbs**和**lsssss**依次对应实体类``Log``属性id、recordId、input、output、exception、creatorId的类型， lsbbbs表示Long、String、Byte[]、Byte[]、Byte[]、String； lsssss表示Long、String、String、String、String、String； 你需要根据项目的实际情况，确定实体类``Log``的泛型参数及日志表列的类型，不局限于以上两种组合。

id设置为Long型，则使用数据库自增； 如果设置为其他类型，不使用数据库自增，则需要实现``IdGenerator``，用于生成主键。

recordId的类型根据你系统中各模块的主键类型决定， 如果都使用Long型自增主键，则recordId的类型设置为Long， 如果有String型主键，则recordId的类型设置为String。 联合主键暂时未考虑在内。

input、output、exception的存储，可以通过java序列化为二进制，也可以转换为json字符串。 因为input、output、exception的大小是不确定，且极有可能很大， 所以存储的长度和溢出后的处理是一个需要慎重考虑的地方。 类库``peacetrue-log-type``提供了``sss``的实现，将input、output转换为json字符串后入库存储， exception直接存储``getMessage()``的返回值，最大长度通过``peacetrue.log.lengths``配置，溢出后会截断。 你可以通过实现``SerializeService``提供自定义的序列化方法； input使用名称为**inputSerializeService**的``SerializeService``； output使用名称为**outputSerializeService**的``SerializeService``； exception使用名称为**exceptionSerializeService**的``SerializeService``。

creator_id则根据你系统的用户主键类型确定。

=== 日志展示

首先，我们需要引入以下类库：

[source%nowrap,maven]
----
<dependency>
    <groupId>com.github.peacetrue</groupId>
    <artifactId>peacetrue-log-controller</artifactId>
    <version>2.0-SNAPSHOT</version>
</dependency>
----

然后，确定``LogQueryDTO``泛化参数的类型。 默认情况下，``recordId``和``creatorId``都会接收到``String``类型的参数值， 如果实际类型不是``String``，我们就无法得到正确的查询结果。 泛型参数的确定，是通过``ConcreteClassArgumentResolver``实现的，
``ConcreteClassArgumentResolver``位于以下类库中：

[source%nowrap,maven]
----
<dependency>
    <groupId>com.github.peacetrue</groupId>
    <artifactId>peacetrue-spring</artifactId>
    <version>1.1.1-SNAPSHOT</version>
</dependency>
----

我们通过以下配置为``LogQueryDTO``指定一个确定了泛型参数的子类：

[source%nowrap,maven]
----
peacetrue.spring.concrete-classes.com.github.peacetrue.log.service.LogQueryDTO=com.github.peacetrue.log.type.ss_LogQueryDTO
----

接下来，我们需要配置请求路径，默认分页查询使用'/logs'， 你可以通过配置``peacetrue.log.urls.query``指定为其他值。

经过以上配置，我们现在可以使用分页查询接口了。
